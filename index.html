GUIzilla

Motive

To understand the principles of network programming and Graphical User Interfaces

Developement

The backend of the project includes a Scala built Client and Server which communicate with each other based on the standard protocol requests and responses. The front end is built on JavaFX with a controller that is responsible for loading new pages with the help of an FXML parser also built in Scala.

Features

The browser can load some online pages but it is primed for pages we created as well, which are complete with forms and dynamic pages (that respond to user actions and input)

Another feature we added to our Browser is the presence of a back button, created using a queue that stores the elements of pages after users have entered their information. This is a feature that is actually not deployed by many browsers. On GUIzilla, users are able to see the web page exactly as they left it, with their inputs and changes present.

ABSH

Motive

To build and understand the BASH scripting language's workings and become comfortable programming in multiprocess environments. ABSH is a play on BASH, it is based on my initials followed by SHell.

Developement

The entire project was built in C. It contains a REPL which reads user input, parses it and checks for the commands in the builtin commands located in the user's bin folder. If these commands are not present, execv is called within a child process. I also had to manage the user's jobs, which was done using a Linked List.

Features

The shell can be run on any terminal and contains extensive functionality, such as job control (users can track processes and move them between the foreground and background at will.

The shell also allows for file redirection (for input and output).

Database

Motive

To create a program that can interact with a database efficiently and securely through the use of threads.

Developement & Features

The database runs on a server which can be accessed by any number of users at a time who can peform actions simultaneously through the use of threads. It is implemented as a Binary Search Tree

The database is made thread safe by the use of mutexes. Rather than locking the entire database when a user requests to manipulate data, the database employs finegrained locking, where only the elements that are crucial do the manipulation are locked.

Search

Motive

Explore writing programs that work with large datasets without losing efficiency or functionality.

Background & Development

Up until Google, most search engines gave the user results based only on the relevance to the query. Today, most search engines have a ranking system which combines the relevance of the page to the query with its relative popularity to the other online pages. This search engine has both implementations.

The entire project was built in Scala. The main challenge of this project was to parse through thousands of pages without wasting the users time waiting 5 minutes for a search result to be displayed. This was done by preparsing every page and creating a dictionary in which we store the relevance of the page to the user's query.

Features

The browser can load some simple online pages but it is primed for pages we created, which are complete with forms and dynamic elements (that respond to user actions and input).

Another feature we added to our Browser is the presence of a back button, created using a queue that stores the elements of pages after users have entered their information. This is a feature that is actually not deployed by many browsers. On GUIzilla, users are able to see the web page exactly as they left it, with their inputs and changes present.
